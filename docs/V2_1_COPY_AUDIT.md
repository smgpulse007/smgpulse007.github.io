# Portfolio V2.1 copy audit

## Default-language change

V2.0 often explains how the portfolio governs its own evidence. V2.1 keeps that rigor in source notes while letting ordinary readers encounter the system, decision, and result first.

| V2.0 pattern | V2.1 treatment |
|---|---|
| `Professional system — sanitized` | `Professional work` in normal reading mode |
| `resume-supported` | source note phrased as `Career record` |
| `public-source` | `Public repository` or named source |
| `private-evidence` | a short local privacy boundary |
| `Synthetic / public-safe` | `Illustrative example` only when the distinction matters |
| headings about proof or honesty | headings about the system, decision, and result |
| `Download résumé` | `Public résumé` |
| `with the labels left on` | maturity and limitations shown quietly in context |

The internal enums remain available to build-time validation but are never printed as default UI copy.

## Approved homepage spine

- Eyebrow: `Senior Applied AI Engineer · Healthcare & Insurance`
- Headline: `I build AI systems that survive real operations.`
- Body: `Claim packets, clinical documents, and event streams rarely arrive ready for a model. I design the contracts, validation, human review, and observability that turn them into accountable workflows.`
- Support: `Agentic workflows · Document intelligence · ML platforms`
- Selected work heading: `The systems behind the outcomes.`

## Ownership rules

- Claims and on-prem stories use first-person ownership for architecture, evaluation, workflow, and review decisions, while keeping employer details private.
- Healthcare analytics uses `helped build and scale` and attaches outcomes to the broader product/program context.
- LLM Steering can use direct ownership because it is a public repository owned by Shailesh.
- Let’s Talk Doc remains team recognition. Individual component contribution is qualified and never presented as sole authorship.

## Denylist

Rendered copy must reject generic AI marketing, legacy Builder language, internal evidence enums, phone numbers, precise location, private paths, placeholder language, and unqualified clinical or production claims.
