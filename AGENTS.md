# Portfolio V2 agent operating rules

Read the Portfolio V2 master prompt, Hostinger production addendum, and autonomous execution playbook before modifying this project. The Hostinger addendum supersedes conflicting hosting, custom-domain, and deployment instructions.

## Product and safety

- Build one evidence-led portfolio. Hostinger is the future canonical host; GitHub is source, CI, public evidence, and no-index mirror.
- Work on `codex/portfolio-v2-hostinger`.
- Keep Astro static-first unless a concrete feature requires a server.
- Do not use Hostinger tools or change Hostinger production, DNS, nameservers, email, billing, subscriptions, ecommerce, VPS, or the Website Builder site without the exact phrase `CUTOVER APPROVED`.
- Do not invent evidence, metrics, awards, contributions, screenshots, code maturity, or confidential details.
- Render final metric values in HTML and keep the site usable without JavaScript.
- Run content, link, accessibility, end-to-end, build, and visual QA checks before calling a branch ready.
