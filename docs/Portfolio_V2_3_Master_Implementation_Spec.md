# 2. Operating principle: outcome-constrained, method-open

Previous specifications gave Codex substantial technical permission but still shaped the output too tightly around a predefined metaphor.

V2.3 changes the operating model.

## Outcomes are constrained

The final product must:

- preserve healthcare and insurance as the primary professional identity;
- restore the complete technical breadth;
- prioritize professional work;
- communicate current production agentic AI;
- use public repositories and demos deeply;
- provide first-glance clarity;
- feel modern, elegant, sci-fi, and memorable;
- use meaningful interaction and animation;
- preserve privacy and evidence;
- remain accessible and performant;
- stage safely.

Codex may decide:

- art direction;
- visual metaphor;
- palette;
- typography;
- grid;
- glass system;
- 2D/3D balance;
- motion library;
- visualization approach;
- component architecture;
- exact information architecture;
- homepage section count;
- framework additions;
- Hostinger-enhanced features;
- standard/mirror fallbacks.

Do not mechanically preserve:

- Systems Observatory branding;
- Signal → Prediction → Product → Context → Agent → Harness;
- V2.2 navigation;
- V2.2 type;
- V2.2 lime palette;
- V2.2 dark/light sections;
- V2.2 card grid;
- V2.2 component templates;
- V2.2 first fold;
- V2.2 page names.

Use V2.2’s research, evidence, route, QA, and infrastructure work where valuable.

Do not use its design direction merely because it exists.
