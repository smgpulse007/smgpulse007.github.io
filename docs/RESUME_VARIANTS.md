# Public résumé variants

## Selected public edition

V2.1 uses a one-page, ATS-readable, privacy-cleared résumé generated by `scripts/generate-public-resume.py`.

The selected composition prioritizes:

- target role and domain within the first scan;
- four career eras with scoped outcomes;
- concise selected engineering work;
- education and credentials without creating an appendix page;
- no phone number or precise location;
- active text links and extractable text;
- enough whitespace to feel intentional without leaving a mostly empty page.

## Rejected public variants

1. **Two-page continuation** — rejected because the second page held only a short project/education appendix and was mostly empty.
2. **Dense one-page skills matrix** — rejected because it optimized keyword volume at the cost of leadership and outcome clarity.
3. **Designed infographic résumé** — rejected because it weakened ATS extraction and duplicated the portfolio’s visual role.

The private application résumé remains outside this repository and is not modified by V2.1.
