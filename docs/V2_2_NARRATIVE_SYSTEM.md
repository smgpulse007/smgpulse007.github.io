# Portfolio V2.2 narrative system

## Core thesis

**Intelligence becomes a system.** The portfolio shows a career-long progression from observable signals to predictions, products, context, agents, and finally execution harnesses. The progression is a change in operating responsibility, not a claim that every project belongs to one linear technology stack.

## Three-part creative model

- **Observatory:** the visual world and first-fold promise. Systems are mapped, instrumented, and inspectable.
- **Evolution:** the navigation and narrative spine: Signal → Prediction → Product → Context → Agent → Harness.
- **Agent system:** the authority language. Model, context, tools, typed state, validation, reviewer authority, and trace are distinct layers.

## Audience sequence

1. Recruiter: role clarity, systems thesis, public-safe outcomes, current frontier.
2. Hiring manager: system index, evolution, operating decisions, limits.
3. Technical reviewer: topology, traces, harness artifacts, primary research, repositories.
4. Collaborator: human origin, recognition, contact, public résumé.

## Evidence grammar

Every system may state **problem / built / changed / proof / limit**. Professional work uses sanitized diagrams and qualified outcomes. Public projects link to exact repository evidence. Synthetic examples are visibly labeled. Unsupported ownership, clinical validation, affiliation, or production claims are prohibited.

## Route roles

- `/` — twelve-scene observatory and five-second identity.
- `/systems/` — canonical professional systems index.
- `/evolution/` — career and abstraction progression.
- `/systems/claims-agents/` — current frontier and bounded agent trace.
- `/systems/meta-harness/` — flagship execution harness.
- `/systems/llm-steering/` — flagship model-behavior laboratory.
- `/lab/` — public systems organized by maturity and limit.
- `/research/` — verified primary-source atlas.
- `/about/`, `/recognition/`, `/resume/` — human context and public proof.

## Voice

Direct, technically literate, specific, and restrained. Explain what a system changes and where authority stops. Avoid portfolio-construction meta-language, generic futurism, or evidence language that reads like an internal audit UI.
